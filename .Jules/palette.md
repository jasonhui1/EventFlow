## 2024-05-18 - Descriptive ARIA Labels in Dynamic Components
**Learning:** `title` attributes alone are insufficient for screen readers, especially in dynamic lists where the visual context (like the item name) isn't inherently read by the screen reader on focus. Icon-only buttons nested inside these items compound the problem if they aren't uniquely identifiable.
**Action:** Always complement `title` with explicit, descriptive `aria-label`s, utilizing dynamic data like item names to ensure unique identification (e.g., `aria-label={"Close " + itemName + " tab"}`). Additionally, for keyboard interactivity in container elements (like custom tabs), always verify `e.target === e.currentTarget` in the `onKeyDown` handler to prevent nested interactive elements (like the close button) from incorrectly triggering the container's action.
