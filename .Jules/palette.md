## 2024-04-07 - Accessible Non-Native Interactive Elements & Dynamic ARIA Labels
**Learning:** When using non-native HTML elements (like `<div>` or `<span>`) for interactive UI components (such as a TabBar), relying solely on `onClick` handlers creates a significant accessibility barrier for keyboard users. Furthermore, static `title` attributes on repetitive or identical controls (like generic close buttons inside tabs) fail to provide sufficient context for screen reader users when multiple instances exist.

**Action:** For non-native interactive elements, always implement an `onKeyDown` handler alongside `onClick`. This handler must trigger on both the 'Enter' and 'Space' keys, check `e.target === e.currentTarget` to prevent event bubbling from child elements, and call `e.preventDefault()` to stop default page scrolling behavior. Additionally, ensure correct ARIA roles (`role="tablist"`, `role="tab"`) and states (`aria-selected`) are applied. For repetitive elements like close buttons, append a dynamic descriptor to the `aria-label` (e.g., `aria-label={"Close " + itemName + " tab"}`) to ensure each instance is uniquely identifiable to screen readers.
