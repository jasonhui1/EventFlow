## 2024-04-02 - Accessible Tab Bar Interactions
**Learning:** `title` attributes are insufficient for screen readers in complex dynamic components like tab bars. Using `aria-label` with dynamic data (like item names) is necessary. Also, when creating complex UI components with nested actions (like tabs with close buttons), it's important to use a focusable `<div>` with `role="tab"`, `tabIndex={0}`, and an `onKeyDown` handler for `Enter` and `Space` key activation, rather than nesting `<button>` elements which violates HTML specs and accessibility guidelines. When implementing the `onKeyDown` handler, ensuring `e.target === e.currentTarget` is vital to prevent nested interactive elements (like the close button) from incorrectly triggering the container's action.
**Action:** Always complement `title` with explicit, descriptive `aria-label`s utilizing dynamic data in list items or tabs. Use focusable divs for tab containers and strictly avoid nesting interactive elements. Verify focus management and correct target bubbling in keyboard handlers.
