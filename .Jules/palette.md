## 2026-04-01 - TabBar Accessibility Improvements
**Learning:** The application uses `<button>` tags inside `<div>` elements with `onClick` for tabs. This structure is somewhat accessible if the div has proper roles and keyboard navigation, but currently it lacks `role="tab"`, `tabIndex`, keyboard event listeners, and proper `aria-labels` on inner buttons. Also, nesting buttons inside other focusable/interactive elements requires care.
**Action:** When creating complex UI components with nested actions (like tabs with close buttons), use a focusable `<div>` with `role="tab"`, `tabIndex={0}`, and an `onKeyDown` handler for `Enter` and `Space` key activation. Ensure you check `e.target === e.currentTarget` to prevent nested interactive elements from incorrectly triggering the container's action.
