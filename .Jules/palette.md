## 2024-05-24 - Accessible Tabs with Close Buttons
**Learning:** When building complex UI components like tabs that contain their own nested actions (e.g., a "Close Tab" button), you cannot nest `<button>` elements as this results in invalid HTML and poor accessibility. Relying solely on `div` elements with `onClick` without proper keyboard bindings breaks keyboard navigation. Furthermore, adding generic `tabIndex={0}` to the container without considering child focus can cause nested elements to trigger the parent's action if not properly delegated.
**Action:** Use a focusable `<div>` with `role="tab"` and `tabIndex={0}` for the outer interactive container, and implement an `onKeyDown` handler (for `Enter` and `Space`). In the `onKeyDown` handler, always check `e.target === e.currentTarget` to ensure nested interactive elements (like the close button) do not accidentally trigger the parent tab's activation behavior. Always supplement visual titles with dynamic `aria-label`s.
