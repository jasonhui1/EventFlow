## 2024-05-18 - Making dynamic UI components fully accessible
**Learning:** Adding complex dynamic UI components (like a custom tab bar) requires explicit ARIA roles (`role="tab"`, `role="tablist"`), states (`aria-selected`), accessible names for visually hidden context (`aria-label`), and robust keyboard support (`Enter` and `Space`). When adding keyboard support `onKeyDown`, it is important to check `e.target === e.currentTarget` to prevent nested interactive elements (e.g. nested tab close buttons) from incorrectly triggering the container's main action.
**Action:** When creating complex UI elements (like dynamic tabs), structure them correctly with full ARIA semantics, descriptive aria labels for icon-only inputs, and implement `e.target === e.currentTarget` verification on keyboard event listeners to prevent unintended nested interactions.
