## 2025-05-29 - O(1) Cache tied to reference for GetEventsByFolder
**Learning:** In `server/dataStore.js`, the `getEventsByFolder` function used an O(N) `events.filter` operation which was a bottleneck. While using a global Map `eventsByFolderCache` and invalidating it manually is one approach, a more robust way to tie cache to data that might be modified externally (e.g. data reloaded from disk) is using a `WeakMap`. By mapping the parsed `events` array reference directly to a built cache using `const eventsByFolderCache = new WeakMap();`, the cache is automatically invalidated when a new data reference is loaded, eliminating stale read risks.
**Action:** When caching derived data based on a list or object, consider using a `WeakMap` with the source object/array as the key to naturally bind the cache validity to the data's reference lifecycle.
