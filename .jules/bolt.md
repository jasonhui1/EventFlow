## 2024-10-24 - Optimization: O(1) Graph Traversal in simulationUtils
**Learning:** In the `simulationUtils.js` module, `getInheritedPrompts` was repeatedly calling `getParentNodes`, which scanned the entire `edges` array `O(E)` for every single node traversal, leading to an overall `O(N * E)` complexity. Additionally, the BFS traversal used `queue.shift()`, creating an `O(N)` operation for every dequeued node. Building a Map locally inside a function called within a loop incurs allocation overhead that defeats the purpose of map caching for small inner arrays.
**Action:** When working with graph traversals in JS, always prioritize hoisting `Map` creation for ID-to-Edge array lookups to the entry point (e.g., `simulateEvent`) and pass it down the call stack to achieve true `O(N + E)` traversal. For BFS traversal queues, use an index read-pointer (`let queueIndex = 0; queue[queueIndex++]`) instead of `.shift()` to maintain `O(1)` time complexity.
