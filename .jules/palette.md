## 2024-04-09 - Accessible Dynamic Tabs
**Learning:** For dynamic components like lists or tab bars, `title` attributes alone are insufficient. We must complement them with explicit, descriptive `aria-label`s that utilize dynamic data (e.g., `aria-label={"Close " + itemName + " tab"}`). Furthermore, custom non-native interactive elements (e.g., a `<div>` acting as a tab) need `role="tab"`, `tabIndex={0}`, and an `onKeyDown` handler (triggering on Enter and Space, with `e.preventDefault()` to stop scrolling, and verifying `e.target === e.currentTarget` to prevent nested element bubbles) to be fully keyboard and screen-reader accessible.
**Action:** When implementing custom tab components or dynamic lists with icon buttons, always apply specific ARIA labels dynamically based on item names, assign proper roles (`tablist` and `tab`), and ensure both Enter and Space keys trigger the main action without interfering with nested interactive elements.
