## 2024-04-07 - Tab Component Keyboard Accessibility & Dynamic ARIA Labels
**Learning:** When implementing custom interactive elements like custom tabs, utilizing `title` attributes alone is insufficient for screen readers. Furthermore, a generic `aria-label` (like "Close tab") applied inside a dynamically mapped component fails to uniquely identify the action context (e.g., *which* tab is being closed), creating an ambiguous experience. Additionally, native `onClick` handlers on `div` elements do not provide keyboard accessibility without explicit `tabIndex={0}` and `onKeyDown` listeners (handling both 'Enter' and 'Space'). Lastly, custom tab components should have proper semantic grouping via `role="tablist"` on the container and `role="tab"` with `aria-selected` on the tab items.
**Action:** Always ensure custom interactive elements have a `tabIndex`, capture `Enter` and `Space` key presses to trigger interactions (with `e.preventDefault()` to prevent page scrolling and `e.target === e.currentTarget` check to prevent bubbling issues), use `role` attributes for structural semantics, and ensure dynamically rendered icon-only buttons include descriptive, item-specific `aria-label`s (e.g., `aria-label={"Close " + eventName + " tab"}`).
