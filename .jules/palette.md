## 2024-05-18 - Dynamic ARIA Labels and Keyboard Access for Custom Tabs
**Learning:** When implementing accessibility for custom tab components using non-native elements (e.g., `<div>`), `role="tab"` and `role="tablist"` are necessary, but dynamic interactive elements like icon-only close buttons require explicitly descriptive dynamic `aria-label`s (e.g., `aria-label={"Close " + eventName + " tab"}`) for screen readers to differentiate them. Furthermore, keyboard accessibility requires an explicit `onKeyDown` handler for 'Enter' and 'Space' keys, ensuring `e.target === e.currentTarget` to prevent nested interactive elements from bubbling up.
**Action:** Always provide descriptive, unique `aria-label`s for dynamic icon-only buttons, and use a robust `onKeyDown` handler for custom interactive containers, checking the target to avoid capturing events from nested buttons.
