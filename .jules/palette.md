## 2024-04-29 - TabBar Accessibility Improvements
**Learning:** The custom TabBar component completely hid open tabs from screen readers due to missing semantic roles (`tablist`, `tab`). Additionally, when implementing `onKeyDown` handlers on custom interactive components that contain nested interactive elements (like a close button inside a tab), checking `e.target === e.currentTarget` is necessary to prevent the nested elements from unintentionally triggering the parent's action via event bubbling.
**Action:** When building custom tab components or similar interactive lists, explicitly add ARIA roles (`role="tablist"`, `role="tab"`) and manage keyboard focus/selection (`tabIndex={0}`, `aria-selected`, `onKeyDown`). Always verify event targets in handlers on parent containers.
