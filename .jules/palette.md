## 2024-05-18 - [Ensure Custom Tabs Are Fully Accessible]
**Learning:** For accessibility in dynamic components (like lists or tab bars), `title` attributes alone are insufficient. We must complement them with explicit, descriptive `aria-label`s that utilize dynamic data (e.g., `aria-label={"Close " + itemName + " tab"}`). Also, when implementing keyboard accessibility for custom interactive components using non-native elements (like `<div>` acting as a tab), we must use an `onKeyDown` handler that triggers on both 'Enter' and ' ' (Space) keys, include an `e.target === e.currentTarget` check to prevent unintended activation from nested interactive elements, and invoke `e.preventDefault()` to prevent page scrolling.
**Action:** When creating custom tabs, always provide `role="tablist"` on the container, `role="tab"`, `tabIndex={0}`, and `aria-selected` on individual tabs, and handle 'Enter'/'Space' key presses explicitly. Always add explicit `aria-label`s to icon-only buttons.
